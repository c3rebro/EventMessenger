Hauptfenster - Konsole

Die Demo Konsole kann verwendet werden um Ihre aktuelle Konfiguration auf Funktionsfähigkeit zu testen. Ohne das Sie dazu ein vernetztes Schließanlagensetup benötigen.

Die EventMessenger Konsole kann 3 unterschiedliche Arten von Ereignissen erzeugen:

  • "Unlocking Events"
    • Ein Transponder wurde an einer vernetzten Schließung G2 Schließung benutzt. Die Schließung kann diesen Zutrittsversuch annehmen ("access granted") oder verweigern ("access denied"). Die Bezeichnungen des Beispiel-Transponders und der Beispiel Schließung sind vorgegeben.
  • "Input Events"
    • Die Geräte "LockNode Radio", "LockNode Cable" sowie der "WaveNetManager-Router mit "IO Funktionalität" besitzen jeweils 3 separate Eingänge. Die EventMessenger Software kann Zustandsänderungen dieser Eingänge als steigende oder fallende Flanke erfassen.
  • "Door Monitoring Events"
    • Die SimonsVoss Schließungen vom Typ "DoorMonitoring" können den aktuellen Zustand der Tür erfassen. Hierzu zählt die Information ob die Tür z.B. "geöffnet", "geschlossen", "verschlossen" oder "sicher verschlossen" ist.

WICHTIG: Solang die Konsole geöffnet ist, kann die EventMesenger Software keine Ereignisse von der Schließanlagensoftware entgegen nehmen.

Bitte Beachten Sie, das "Unlocking Events" bzw. "Transpondernutzungsereignisse" nur für G2 Schließungen ab einem Firmwarestand größer oder gleich 2.3.10 verfügbar sind.
