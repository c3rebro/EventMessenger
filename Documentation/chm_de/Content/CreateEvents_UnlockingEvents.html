Ereignisse anlegen - Transpondernutzungsereignis

 

Um auf die Verwendung eines Transponders an einer vernetzten Schließung reagieren zu können, muss der EventMessenger die Bezeichnungen und Seriennummern der betroffenen Transponder und Schließungen kennen.

Es wird immer transponderweise geprüft.

Das bedeutet, das Sie immer eine Person bzw. einen Transponder anlegen und eine oder mehrere dazugehörige Tür(en) bzw. eine oder mehrere dazugehörige Schließung(en).


Bitte verfahren Sie wie folgt:

  1.  Legen Sie zuerst immer die Schließung(en) an.
    1. Setzen Sie den Haken "Türbezeichnung und/oder Seriennummer berücksichtigen" wenn nicht pauschal auf alle Schließungen der Schließanlage reagiert werden soll.
    2. Vergeben Sie einen Türnamen und/oder eine Seriennummer für jede Tür auf die reagiert werden soll.

      HINWEISE:
      - Groß- und Kleinschreibung wird nicht unterschieden.
      - Es wird immer nur auf das Vorhandensein der eingegebenen Daten in den von der LSM.Basic.Online übermittelten Daten geprüft.

      Beispiel:
      Wenn Sie als Türbezeichnung "Eingang" eingeben und die Seriennummer leer lassen, wird bei jeder der folgenden von der LSM.Basic.Online an den EventMessenger übermittelten Türbezeichnungen eine positive Übereinstimmung erkannt:

      Eingang Nord; Haupteingang; Eingangsbereich
    3. Klicken Sie auf "Hinzufügen" um die Schließung anzulegen.
  2. Definieren Sie nun den Transponder der das zu erfassende Ereignis auslöst. Gehen Sie bei der Definition des Namens des Transponderbesitzers bzw. der Transponderseriennummer analog zur Definition der Schließungen (Punkt 1.2) vor.

    Beachten Sie bitte, das Sie auch mehrere Transponder gleichzeitig prüfen können, indem Sie die Transpondernamen direkt in das Feld eintragen.

    Beispiel:
    Wenn Sie Herrn Schmid und Frau Müller erfassen möchten, tragen Sie als Transpondernamen "Schmid; Müller" ein. Bitte vergessen Sie nicht, auch die Seriennummern der Transponder mit anzugeben, da es bei vorhandensein mehrerer Personen mit dem Namen "Schmid" oder "Müller" ansonsten zu falsch-positiven Übereinstimmungen kommen kann.
  3. Wählen Sie unter "Ereignis" aus, ob nur bei verweigerten, gestatteten oder beiden Zutrittsversuchen eine Reaktion erfolgen soll.
  4. Mit einem Klick auf "Speichern und Beenden" wird das Ereignis angelegt.

Bitte Beachten Sie, das dieses Ereignis nur für G2 Schließungen ab einem Firmwarestand größer oder gleich 2.3.10 verfügbar ist.
